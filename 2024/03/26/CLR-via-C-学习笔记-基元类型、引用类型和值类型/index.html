CLR via C# 学习笔记-基元类型、引用类型和值类型

Created2024-03-26Updated2024-03-27

1.编程语言的基元类型

编辑器直接支持的数据类型称为基元类型（primitive type）。基元类型之间映射到Framework类库（FCL）中存在的类型。包括：sbyte、byte、short、ushort、int、uint、long、ulong、char、float、double、bool、decimal、string、object、dynamic。

C#只有在“安全”的时候才允许隐式转换，也就是精度或数量级不丢失的情况。否则需要显式转换。

C#默认对溢出是不检查的，也就是溢出后变成负值或很小的值，但是可以使用checked()进行检查，如果溢出就抛出OverflowException异常。

2.引用类型和值类型

引用类型总是从托管堆分配，C#的new操作符返回对象的内存地址–即对象数据的内存地址。首先有四个事实：

  1. 内存必须从托管堆分配
  2. 堆上分配的每个对象都有一些额外成员，这些成员必须初始化
  3. 对象中的其他字节总是设为零
  4. 从托管堆分配对象时，可能强制执行一次垃圾回收

与之对应的是值类型，值类型的实例一般在线程栈上分配（虽然也可作为字段嵌入引用类型的对象中），值类型变量中包含了实例本身的字段，所以操作实例中的字段不需要提领指针，值类型的实例也就不受垃圾回收器控制。

一个比较简单区分值类型和引用类型的方式就是，任何称为“类”的都是引用类型。所有值类型都称为结构或枚举。比如:int就是struct Int32。而进一步研究，就会发现，结构都是抽象类型System.ValueType的直接派生。.net文档

CLR和所有编程语言都给予枚举特殊待遇。

虽然值类型不会申请堆上内存，但是我们也不能什么都去使用值类型，因为值类型在传递、返回的时候需要重新拷贝，如果结构体过大，还不如使用引用类型传一个指针呢。

注：CLR会自动重新排序结构体中的值的存储顺序，所以在和Native代码交互的时候，如果必须使用结构体，需要使用StructLayoutAttribute将结构锁定，不然会导致2端结构体内数据结构不一致导致混乱。

3.值类型的装箱和拆箱

许多时候，需要获取对值类型的引用，这就需要装箱和拆箱操作了。

3.1.装箱

比如：

struce Point{int x,y}, ArrayList points;

ArrayList的Add方法为public virtual Int32 Add (Object value);传递的参数为Object 也就是指针，也就是Add获取托管堆上的一个对象的引用作为参数。这时Point就要从值类型变成引用类型，也就是装箱：

  1. 在托管堆中分配内存。分配的内存量是值类型各字段所需要的内存量，还要加上托管堆所有对象的2个额外成员（类型对象指针和同步块索引）所需要的内存量。
  2. 值类型的字段复制到新分配的堆内存
  3. 返回对象地址。

目前我们经常使用的List<T>等泛型类，已经不在有装箱拆箱的操作了。并且类型安全也进一步提高了。

3.2.拆箱

比如我们想获取ArrayList的第一个元素：Point p = (Point) a[0];这时就会发生拆箱：

  1. 获取已装箱对象中的各个字段的地址
  2. 将字段包含的值复制到基于栈的值类型实例中

注：拆箱不是直接将装箱过程倒过来。拆箱的代价比装箱低得多。拆箱其实就是获取指针的过程，该指针指向包含在一个对象中的原始值类型(数据字段)。其实，指针指向的是已装箱实例中的未装箱部分。所以和装箱不同，拆箱不要求在内存中复制任何字节。知道这个重要区别之后，还应知道的 一个重点是，往往紧接着拆箱发生一次字段复制。
